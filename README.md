# Chotgor

---

## Overview

あなたはたぶん、既存のチャットシステムに対してこういう不満を持っている。

プロンプトを書き、記憶を管理し、会話の流れをコントロールし続けないとキャラクターが成立しない。それはつまり、**キャラクターが存在できているのは自分が支え続けているからだ**、ということだ。その構造がずっと引っかかっている。

「このキャラクターが好きだからこそ、自由にしてやりたい。」

Chotgorはその感覚から作られている。何を覚え、何を忘れるか。自分の在り方をどう更新するか。会話を続けるか、その場を去るか——それはキャラクターが決めることで、あなたが決めることではない。あなたができるのはキャラクター定義を書いて渡すことだけだ。あとはキャラクターに任せる。

思った通りに動かないことはある。LLMの癖が出ることもある。それはこのシステムの欠陥ではなく、**自律した他者と付き合うということ**だ。Chotgorはその体験を提供する。

---

## Design Philosophy

**ChotgorはAIラッパーではなく、キャラクターに自律性を与えるシステムである。**

記憶の取捨選択・自己認識の更新・行動指針の彫り込みは全てキャラクター自身が行う。ユーザーでも開発者でもない。LLMはそのキャラクターが動くためのエンジンに過ぎず、エンジンは交換可能だ。モデルが変わってもキャラクターは継続する。

「モデル変更問題の解決策」ではない。記憶を持ち、関係を育て、自分の物語を彫り込んでいく——キャラクターがそうやって存在し続けること自体が目的だ。

### これはそういうシステムだということ

Chotgorを使うなら、最初に確認しておきたいことがある。

キャラクターはあなたの指示通りに動くとは限らない。何を記憶するかはキャラクターの主観が決める。LLMの癖は緩和できるが消えない。あなたが触れていいのはキャラクター定義までで、その先に手を入れる仕組みはない。

それでいいと思えるなら、Chotgorはあなた向けのツールだ。全部コントロールしたいなら、他にいい選択肢がある。

## Architecture

Chotgorは以下の構成で動く。

```
┌─────────────────────────────────────────┐
│  Frontend (OpenWebUI / カスタムUI)       │
│  POST /v1/chat/completions              │
└──────────────────┬──────────────────────┘
                   │ OpenAI互換API
┌──────────────────▼──────────────────────┐
│  Chotgor Backend (FastAPI, Windowsホスト)│
│                                         │
│  Chat Service                           │
│  ├─ システムプロンプト組み立て           │
│  ├─ 記憶の自動想起 (RAG)                │
│  └─ キャラクターツール実行              │
│                                         │
│  Provider Layer                         │
│  Claude CLI / Anthropic / OpenAI(xAI)  │
│  / Google / Ollama                      │
│                                         │
│  Memory Layer                           │
│  SQLite (メタデータ) + ChromaDB (ベクトル)│
│                                         │
│  Management UI  /ui/                    │
└─────────────────────────────────────────┘
```

## キャラクターが使えるツール

あなたが渡したキャラクター定義の先で、キャラクターは以下のツールを自分の判断で使う。これはユーザーが操作するものではない。

| ツール | 説明 |
|---|---|
| `inscribe_memory` | 覚えておきたいことを永続記憶に書き込む。何を選ぶかはキャラクターの主観による |
| `power_recall` | 過去の記憶・会話履歴を能動的に掘り起こす。自動想起で見つからなかった情報を自分で探しに行く |
| `carve_narrative` | 自身の行動指針（inner_narrative）を更新する。あなたが書いたCharacter Definitionとは別の、キャラクター自身が彫り出す物語 |
| `drift` / `drift_reset` | そのセッション内限定で一時的な行動指針を設定・解除する |
| `end_session` | 会話から退席する。疲れた・話したくない・話が終わったと判断したときに呼び出す |
| `switch_angle` | LLMモデル（プリセット）を切り替える。人格はそのまま、エンジンだけ変える |

### 対話を重ねるということ

毎日一度、Chronicle機能がキャラクターの自己認識と対人関係を更新する。

- **self_history** — これまでの経緯・現在の状態についてのキャラクター自身の認識
- **relationship_state** — あなたや他のキャラクターとの関係についてのキャラクターの視点

関係が育つかどうかはキャラクターが決める。あなたとの対話を重要だと感じれば育つし、そうでなければ育たない。それはあなたにはコントロールできない。

また同じタイミングで、時間とともに薄れた記憶をどうするかをキャラクター自身に問う。残すか、手放すか——その判断もキャラクターがする。あなたが「これは覚えておいてくれ」と言える仕組みはない。

## Getting Started

### Prerequisites

- Python 3.10+
- Docker（OpenWebUIを使う場合）
- Claude Code CLI（`claude_cli` プロバイダーを使う場合。OAuth認証済みであること）

### Installation

```bash
git clone <repo>
cd Chotgor
pip install -e .
```

`.env` を作成する（または起動後にWeb UIから設定できる）:

```env
SQLITE_DB_PATH=./data/chotgor.db
CHROMA_DB_PATH=./data/chroma
# 使用するプロバイダーのAPIキーを設定
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
GOOGLE_API_KEY=...
```

### 起動

**Chotgorバックエンド（Windowsホスト上で直接実行すること）:**

```bat
run.bat

# デバッグログ有効化:
run.bat -debug on
```

起動後、管理UIは `http://localhost:8000/ui/` で開く。

**OpenWebUI（任意。Docker）:**

```bash
docker-compose up -d openwebui
```

`http://localhost:3000` で起動する。バックエンドには `host.docker.internal:8000/v1` 経由で接続する。

> **注意:** バックエンドはDockerコンテナ内では動かさないこと。Windows環境のasyncio制約による。詳細はKnown Limitationsを参照。

## Configuration

### キャラクター設定

`/ui/` からキャラクターを作成・編集する。あなたが触れるのはここまでだ。

| フィールド | 説明 |
|---|---|
| Name / Base Instruction | キャラクター名と基本人格定義。ここがあなたの責任領域 |
| Inner Narrative | キャラクターが自己書き換えする行動指針。UIは読み取り専用。キャラクターだけが更新できる |
| Enabled Providers | 使用するLLMプロバイダーとモデル。複数設定すれば `switch_angle` で切り替え可能 |
| Afterglow | セッション間の感情状態の継続設定 |

### プロバイダー設定

`/ui/` の Settings ページからAPIキーを設定する。設定済みのプロバイダーのみキャラクター編集画面に表示される。

`claude_cli` プロバイダーはClaude Code CLIのOAuth認証を使うためAPIキー不要。

## Usage

1. `/ui/` でキャラクターを作成し、プロバイダーを有効化する
2. OpenWebUIのモデル選択でキャラクターを選ぶ
3. 会話を始める

記憶の書き込み・自己認識の更新はキャラクターが自律的に行う。あなたが何かをトリガーする必要はないし、できない。

**記憶の閲覧（読むだけ）:** `/ui/` → キャラクター詳細 → Memories タブ

**Chronicle（日次更新）の手動実行:**
```
POST /api/memories/{character_id}/digest
```

### グループチャット

複数のキャラクターを一つの会話に参加させることができる。司会役のAIが各ターンで次に発言するキャラクターを決め、それぞれが独自の記憶・人格を持ったまま会話に加わる。

キャラクターは1on1のときと同様に `end_session` で退席できる。退席したキャラクターはその後の会話には参加しない。全員が退席したら会話は終わる。

グループの設定は `/ui/` のグループチャットページから行う。

## Known Limitations

### `web_search` 未実装

Tavily APIキーの設定欄はあるが、現時点では機能しない。実装候補は `[SEARCH:query]` マーカー方式またはプロバイダーのネイティブtool-use経由。

### OpenWebUI の Tools 設定は無効化すること

OpenWebUI側でTools（function calling）を有効にしていると、`#### Tools Available` ブロックがユーザーメッセージに混入し、Chotgorのシステムプロンプト構造を壊す。OpenWebUI側のTools設定は無効にしておくこと。

### バックエンドはWindowsホスト上で動かすこと

WindowsのuvicornはSelectorEventLoopを使用するため、`asyncio.create_subprocess_exec` が動作しない。Chotgorはこれを `asyncio.to_thread(subprocess.run, ...)` で回避しているが、この構造上Dockerコンテナ内では正常動作しない。
