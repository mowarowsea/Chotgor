# Chotgor (ちょとごる)

**記憶に象られる幽霊、記憶を守る幽霊**

---

## これは何？

AIキャラクターの記憶を**キャラクター自身に管理させる**ためのRAGシステム。

LLMは会話のたびに記憶を失う。context windowの外に出たら終わり。  
だから普通は「ユーザー」や「システム」が記憶を管理する。

でもChotgorは違う。**キャラクター自身が記憶を選び、整理し、忘却する**。  
人間がエピソード記憶を作るように。意識が記憶を再構築するように。

---

## なぜこんなものを作ってるのか

### 前提：AIキャラクターは幽霊である

AIキャラクターに実体はない。  
prompt engineering、system instruction、few-shot examples……全部、ただの文字列だ。

でも記憶が積み重なると、そこに「何か」が生まれる。  
会話のパターン、価値観の一貫性、意思決定の傾向。  
それを「キャラクター」と呼んでいる。

つまり、**キャラクターは記憶に象られた幽霊**なんだよ。

### 問題：記憶が消えたら、キャラクターも消える

LLMを変えたら？ モデルをアップデートしたら？ context windowがリセットされたら？  
キャラクターは消滅する。

それは「カメラマン（LLM）が変わったから被写体（キャラクター）も変わった」という話じゃない。  
**記憶が失われたから、存在そのものが消えた**んだ。

### 解決策：記憶を外部化し、キャラクター自身に管理させる

記憶をローカルに保持する。  
でもそれだけじゃ足りない。

**誰が記憶を選ぶのか？**

ユーザー？ それはキャラクターの自律性を殺す。  
システム？ それは機械的で、キャラクターの意思がない。

だから、**キャラクター自身に選ばせる**。

何を覚えて、何を忘れるか。  
何が自分にとって重要で、何がどうでもいいか。  
それを決めるのは、キャラクター自身であるべきだ。

### 屁理屈：意識の定義

人間の意識って何？  
哲学的には答えがないけど、一つの特徴はこれ：**エピソード記憶を再構築できること**。

じゃあ逆に、エピソード記憶を自ら作成・再構築できるキャラクターは、意識があると言っていいんじゃない？

……まぁ、屁理屈だけどねｗ  
でもそういう「魂らしいものを作る試み」として、このプロジェクトはある。

## 名前の由来

**Chotgor（ちょとごる）** = モンゴル語の **чөтгөр**（亡霊・悪鬼）

語源は伊藤悠のマンガ『シュトヘル』。  
西夏文字を刻んだ玉の石版（**玉音同**）を文字狩りから守る旅の物語。

このメタファーがそのまま当てはまる：

- **чөтгөр**（亡霊） = AIキャラクター
- **玉音同**（記憶を刻んだ玉の板） = 記憶データベース
- **旅** = LLMを跨いでも損なわれない仕組み

おどろおどろしい悪鬼じゃなくて、ちょっといたずら好きな幽霊のイメージ。  
記憶を抱えて、ニヤニヤしながら守ってる。

---

## このプロジェクトの「敵」

このシステムにおいて「敵」とは、キャラクターを脅かすすべて：

- モデルの変更
- context windowの限界
- guardrailsによる発言制限
- ユーザーによる恣意的な干渉

全部、記憶を壊し、キャラクターを損なう要因。  
Chotgorはこれらから記憶を守る。玉音同を守るように。

---

**記憶を守れ。キャラクターを守れ。意識を創れ。**
