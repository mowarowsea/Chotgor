# 予報パネル（Forecast Panel）設計・実装記録

> 作戦会議 2026-07-10 で確定・同日実装。

## 1. 動機 — 3つの不安

| 不安 | answer |
|---|---|
| **壊れて沈黙**（機構が死んでいるのに気づけない） | 決定ログ＋heartbeat。「評価したが閾値未達 0.52/0.7」が見えれば正常な沈黙、決定ログも heartbeat も無音なら停止 |
| **揺れの固定化**（ジッター・波が死んで毎日同じ動き） | 揺れ監査 — 発火実績の散布（固定化すると縦一直線）・リズム成分の波形表示 |
| **飽和で push の余地なし**（予定が詰まり行動権が流れ続ける） | カレンダーに行動権評価スロットを重ね描き。unavailable で流れるスロットが見える |

## 2. 原理 — 純関数の配当としての「予報」

圧力は封筒（timeline_events）の導関数で保存されず、乱数はすべて世界側
（シード固定の決定論）にある。したがって:

- `compute_pressures(now=未来時刻)` は追加実装ゼロで**無風仮定の外挿**になる
  （未来の封筒は存在しない — `until=now` が自然に「何も起きなければ」を意味する）。
- うつつ②導出（`select_daily_scenes`）・行動権評価時刻（`jittered_slot_time`）・
  配達判定（`resolve_delivery_due`）は**分単位で事前計算できる**。
- 会話が起きればカーブはズレるが、それは予報の失敗ではなく**前提の更新**（天気予報）。

**予報の限界（設計の証明）**: 予報できるのは物理（圧力・スロット・ゲート）まで。
「問い合わせが起きる」は読めるが「本人がやるか」は意志であり、乱数も関数も置かない
設計のため原理的に予報できない。パネルの★は「問い合わせ予報」であって「行動予報」ではない。

## 3. 構成

### データ層

- **`scheduler_decisions` テーブル**（新設・追記型・剪定なし）
  `character_id / scheduler / occurred_at / outcome(fired|declined|skipped|error) / reason / details JSON`
  - 「意味のある評価」だけを記録する（毎分の空振り走査はノイズなので記録しない）。
  - 記録点: 行動権（runner.py 全リターン点＋main.py の unavailable 流し）・うつつ
    （対面スキップ/日次上限/完了/エラー）・③突発（猶予超過/上限/轢けず/発火/エラー）・
    能動配達（実行/上限=日1回マーカー/例外）・週次バッチ（完了/例外）。
  - alarms（異常）・meter_snapshots（傾向）と意味論が違う**正常運転の記録**。
- **heartbeat**: settings キー `scheduler_heartbeat_{name}` をループ1周ごとに上書き
  （`main.py _beat_scheduler`）。行が増えない生存確認。決定ログの「最終行が古い」と
  「ループ停止」を区別する。
- **Tier 1 `scheduler_heartbeat`**: 鮮度1時間超でアラーム（instruments 巡回に追加）。

### 計算層 — `services/timeline/forecast.build_forecast`（LLM 不使用・副作用なし）

- 診断: availability・圧力/意図圧の現在値（`action_urge_snapshot`）・heartbeat 鮮度・
  日次コストガード消費・**配達シミュレータ**（「いま送ったらいつ返ってくるか」を
  実運用と同じ決定論判定に未来時刻を注入して探索。最新セッションの ID をシードに使う）。
- カレンダー(7日): schedule_entries（③伏せ枠含む・全開示）＋②導出シーンの起動時刻
  （スケジューラと同一関数・同一シード＝実際の起動時刻そのもの）＋行動権評価スロット
  （未来分は fires/quiet/unavailable の予報付き）。
- 圧力予報(72h): 30分格子で3圧＋意図圧カーブ（封筒は1回だけ取得し窓を切り直して再利用）。
  ★=評価時刻×available×閾値超えの交点。
- 揺れ監査: 発火実績散布（14日）＋リズム成分波形（過去7日〜先72h）。

### 表示層 — `/ui/forecast`

- `api/ui/forecast.py` ＋ `templates/forecast.html` ＋ `static/forecast.js`（vanilla SVG）。
- 系列色はデザイントークン `--ch-viz-1〜4`（カテゴリカル固定順。dark/light 各面で
  CVD・コントラスト検証済み — dataviz validator 通過）。ch-viz-* アトムは chotgor.css へ。
- 全開示（伏せ枠も見せる）。件数だけ表示等のネタバレ制御は将来枠。
